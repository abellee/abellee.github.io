golang的bytes

前几天在弄golang的验证码的时候，找了一个现成的类 captcha
这个类很方便的解决了golang的验证码问题，但事实上是，我又很费劲的去熟悉了一番golang的images库，
也啃了一下这个类的代码，受益良多，让我了解了文字在内存中的存在形式。
但有个问题是，这个类对验证码的保存并不是使用session，
因为它是一个是独立的工具类，不依赖于任何框架，但我希望用session来保存验证，
所以无奈只能改！

我用的是Revel这个框架，Session中只能保存字符串，而这个类生成的验证码是[]byte形式的
并且里面的每个元素都是一个rune 也就是uint32，所以我没办法直接转成string
所以用for转成了string，但因为我输出图片的方式是直接改头信息，内容就无法直接保存到Session里去，
所以我只能是在进入页面的时候就生成验证码，并保存到Session里去，
然后在生成验证码图片的请求里直接去取Session里的值，再把这个值转回到[]byte形式，交给这个类处理。
所以在进入页面的时候的处理如下：

    num := captcha.RandomDigits(6) //获取验证码图片上的6个随机的0-9之间的数字
    s := ""
    for _, v := range num{
        s += strconv.Itoa(int(v))
    }

这样就可以把s存到Session中去了，在for里面，我直接用strconv.Itoa强制把v转成int型，
按golang本身的规则，可能是会出问题的，因为v是uint32的，但我也实在想不出来，什么情况下int跟uint32会不同，
而我使用的环境基本上不会发现差异，所以就放心的转了。

然后就是获取验证码的地方了， 

    code := strings.Split(c.Sessioin["cap_code"], "")
    buff := byes.NewBuffer([]byte{})
    for _, v := rangecode{
        val, _ := strconv.Atoi(v)
        buff.WriteRune(rune(val))
    }

然后再取buff.Bytes()就是这个类所能使用的[]byte了。
