EaselJS(CreateJS)屏幕适配

EaselJS(CreateJS)本身不进行任何的屏幕适配工作，所以必须要我们自己解决。google了老半天也没找到合适的解决方案，老外使用屏幕适配的场景貌似跟我们国内的有所不同，事实上我也拿着那些代码试了各种情况，都不合适，而EaselJS官方所讨论的，只是根据设备像素比（devicePixelRatio）的一种计算方式，但事实上，我们想要解决的毕竟只是显示的问题，所以，我们只要拿设备的屏幕尺寸，更确切来讲，应该是容器的尺寸，来跟我们的设计尺寸进行计算，就可以得到最终想要的效果了。

这里要解释一下其中的一些关键词：

屏幕尺寸：也就是你设备屏幕的物理尺寸，比如iPhone 5s，他的物理尺寸是宽320，高568。其实在无线端，屏幕尺寸跟容器尺寸是一样的，毕竟用户是无法手动去修改浏览器大小的，当然，hybrid方式开发的就不同了。

屏幕分辨率：这与物理尺寸不同，要考虑到设备dpi问题，例如：iphone 5s是retina屏，他们的设备像素比（devicePixelRatio）是2，但屏幕物理尺寸是宽320 高568，所以它实际分辨率是宽640 高1136

容器尺寸：就是包裹canvas的容器的尺寸，如果你直接放在body里面，就是浏览器的尺寸。如果是在某个div里的，就是div的尺寸。

设计尺寸：这是设计师在设计的时候定的尺寸，其实也就是项目一开始制定的首选设备的分辨率。例如：设计是按iphone 5s的分辨率，也就是宽640，高1136

好了，直接上代码，代码很简单：

var canvas = document.getElementById("gameStage");
var designWidth = 640; // 设计宽度为640
var designHeight = 1136; // 设计高度为1136
var viewWidth = document.documentElement.clientWidth; // 获取浏览器可视区域宽度
var viewHeight = document.documentElement.clientHeight; // 获取浏览器可视区域高度
var scale = viewWidth / designWidth; // 缩放因子
canvas.width = designWidth;
canvas.height = viewHeight / scale;
canvas.style.width = viewWidth + "px";
canvas.style.height = viewHeight + "px";

var stage = new createjs.Stage(canvas);

OK，这就完成了适配工作了。
上面的代码是全屏的方案，如果你的项目放在页面的某个div中，你只要改一下viewWidth跟viewHeight为div的宽、高就可以了。当然，这种情况我没试过，需要你去试验一下。

还有一个跟屏幕适配相关的问题，也就是resing。Adobe是EaselJS(CreateJS)其中的一家赞助商，而框架本身的API设计，更是极大的模仿了as3的接口，所以这个时候，跟我一样曾经是as3开发者的同学们，就想起了Stage的onResize，但不幸的是，框架本身也没有提供这方面的支持，唉，不知道以后的版本会不会提供，但现在我们还是得自己干，你可以自己写个类继承自createjs.Container（as3对应DisplayObjectContainer），然后在里面添加一个resize的方法，或者监听onResize事件，来处理容器尺寸改变的情况。而容器尺寸改变的事件，使用window.onresize即可。至于什么用，那就要看你怎么发挥了。
