关于beego中访问crossdomain

在ActionScript中，访问服务端的静态资源的时候，比如图片、文件等的时候，都需要服务端指定一个crossdomain.xml，
以告诉flashplayer其访问权限，以前服务端用的都是Apache之类的，估计这部分工具已经做掉了，但现在用golang，貌似什么都要自己来，折腾了好几个小时，来回在beego官方文档里找，也没有关于这方面的说法，唯一跟我们的需求相关的，就是关于静态文件的处理，测试了一下，貌似只能显示图片，可能是beego中间层做了什么，只能怪自己笨了。

后来看到beego文档中关于路由的说明，这还得归功于google啊，哈哈，google把xml相关的都高亮了出来，才让我看到beego官方文档中的提到，可以把**.xml路由到某个controller，这时候觉得自己太SB了，当初beego不支持这种类似于twitter api方式的路由，我还去github上开了issue跟作者纠结老半天，现在竟然卡在这个问题上几个小时。不过万幸问题得以解决，但不知道有没有官方的的做法，接下来就说一下我的解决方案吧。

先在beego的router.go中：

beego.Router("/crossdomain.xml", &controllers.CrossDomainController{})

crossdomain.go中

package controllers

import (
    "encoding/xml"
    "github.com/astaxie/beego"
)

type CrossDomainController struct {
    beego.Controller
}

type AllowAccessFromNode struct {
    XMLName xml.Name `xml:"allow-access-from"`
    Domain  string   `xml:"domain,attr"`
}

type flashPolicyXml struct {
    XMLName     xml.Name `xml:"cross-domain-policy"`
    Domain      string   `xml:"domain,attr"`
    Port        string   `xml:"to-ports,attr"`
    AllowAccess AllowAccessFromNode
}

func (this *CrossDomainController) Get() {
    v := flashPolicyXml{}
    v.Domain = "*"
    v.Port = "*"
    v.AllowAccess.Domain = "*"
    this.Data["xml"] = &v
    this.ServeXml()
}

这样FlashPlayer就不会再报安全策略问题了。
不过这样返回的xml，FlashPlayer还是会报关于master-only的问题，虽然这是个”警告”，但你可以通过设置返回的xml中的permitted-cross-domain-policies来解决这个问题。

OK！
