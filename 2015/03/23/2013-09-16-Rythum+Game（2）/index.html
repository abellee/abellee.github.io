Rythum Game(2)

这些天找了全世界的各种资料，

也试着敲了些代码研究数据，

感觉可能是卡在哪一步了。

首先需要知道几个名词，

bpm Beat Per Minute，也就是每分钟节拍数，通俗点讲，这说明音乐的速度。而到程式里后，因为要跟音乐的时间相关联，所以一般会用以下方法转成beat per millisecond

    60 * 1000 / bpm

Sound.extract() 这个方法能把一首音乐的全部原数据提取到一个ByteArray，也就是说 这样我们就能用它来提前处理好数据，假设有一张画布，宽是4个一定宽度的格式的宽，高是一定高度，而这个高度代表了整首曲子的总时间，而各个note点根据时间点散布在这个画布上，然后根据bpm往下拉画布。

以下是wiki上关于BPM的一句引用

例如上图（Quarter note with upwards stem.svg＝120），120是其BPM值，表示每分钟演奏120个四分音符，即每个四分音符的长度等于1分钟除以120等分＝0.5秒（如上图拍子记号是4/4拍，1小节就是0.5秒乘4拍＝2秒长）。BPM的数值越大代表越快的速度。

比如bpm为120，那就是说一拍为0.5秒，首先把歌曲总时间转为画布高度，计算出1像素为歌曲多少时间，再计算这段时间里有多少拍，假设1像素等于0.1秒，也就是说5像素为一拍，即，画面每隔5像素添加一个note。

大家玩过很多节奏类的音乐游戏应该发现，很多音乐，在开头的时候是不会有note掉落的，也就是说，如果我们完全按照以上数据来添加note，就无法达到这样的效果。那可以设定一个threshold，也就是阈值，通俗的讲，就是从所有的声音数据中取一个平均值，高于该值，则添加note，低于该值就不添加，那一首歌曲的note数就等于>threshold的数量。

在音樂理論中，速度（tempo）是一個非常重要的音樂元素，它給與了音樂作品的速度，影響了作品的情感與演奏難度。
音樂速度一般都會注於在一個樂曲的開始位置，現代音樂都以「拍每分鐘」（beats per minute，省略為 BPM）作單位。
例如節奏BPM＝60，表示1分鐘有60個四分音符的長度, 拍子記號是4/4拍, 那每一個4音符的長度＝1分鐘除以60等分＝1秒
1小節4拍的時間長度 ＝ 1秒 X 4拍＝4秒的時間值
例如節奏BPM＝120，表示1分鐘有120個四分音符的長度, , 那每一個4音符的長度＝0.5秒(有1分鐘除以120等分)
1小節就4拍的時間長度 ＝ 0.5秒 X 4拍＝2秒時間值
BPM 的數值越大，拍子速度就越快。
