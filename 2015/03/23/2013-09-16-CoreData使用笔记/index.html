CoreData使用笔记

作一下记录，以供以后reload - -！！！

以下是通过一个command line tool的project来说明CoreData中的使用的

command line tool是创建自OS X的Application

我使用mysql的库，直接连接到mysql，把一些死数据读到CoreData的sqlite文件里，

然后再把这个sqlite文件移至ios项目里，以达到部分数据与应用捆绑发布的目的。

CoreData的基础使用方法在ios或mac os x里都是一样的 所以你可以无缝的在这两种工程里来回迁移，只需要做部分调整，下面会提到。

创建好command line tool工程后 把除了main方法外的所有方法删除

在main上面添加以下两个方法：

  static NSManagedObjectModel *managedObjectModel()
  {
      static NSManagedObjectModel *model = nil;
      if (model != nil) {
          return model;
      }
      NSString *path = @"****"; //此处的*号替换成你xcdatamodeld的名称
      path = [path stringByDeletingPathExtension];
      NSURL *modelURL = [NSURL fileURLWithPath:[path stringByAppendingPathExtension:@"momd"]];
      model = [[NSManagedObjectModel alloc] initWithContentsOfURL:modelURL];
      return model;
  }

  static NSManagedObjectContext *managedObjectContext()
  {
      static NSManagedObjectContext *context = nil;
      if (context != nil) {
          return context;
      }

      @autoreleasepool {
          context = [[NSManagedObjectContext alloc] init];
          NSPersistentStoreCoordinator *coordinator = [[NSPersistentStoreCoordinator alloc] initWithManagedObjectModel:managedObjectModel()];
          [context setPersistentStoreCoordinator:coordinator];
          NSString *STORE_TYPE = NSSQLiteStoreType;
          NSString *path = [[NSProcessInfo processInfo] arguments][0];
          path = [path stringByDeletingPathExtension];
          path = @"****"; //此处的*号替换成你xcdatamodeld的名称
          NSURL *url = [NSURL fileURLWithPath:[path stringByAppendingPathExtension:@"sqlite"]];
          NSError *error;
          NSPersistentStore *newStore = [coordinator addPersistentStoreWithType:STORE_TYPE configuration:nil URL:url options:nil error:&error];
          if (newStore == nil) {
              NSLog(@"Store Configuration Failure %@", ([error localizedDescription] != nil) ? [error localizedDescription] : @"Unknown Error");
          }
      }
      return context;
  }

然后main里面：

  int main(int argc, const char * argv[])
  {
      @autoreleasepool {
          // Create the managed object context
          NSManagedObjectContext *context = managedObjectContext();
          mysqlDB* mysql = [[mysqlDB alloc] init];
          [mysql connectHost:@"127.0.0.1" connectUser:@"mysql帐号" connectPassword:@"mysql密码" connectName:@"连接的数据名称"];
          [mysql query:@"select * from 数据库名称"];
          NSMutableArray* results = [mysql fetchAllRows:[mysql store_result]];
          for (NSMutableArray* row in results) {
                    //.....
          }
          // Custom code here...
          // Save the managed object context
          NSError *error = nil;
          if (![context save:&error]) {
              NSLog(@"Error while saving %@", ([error localizedDescription] != nil) ? [error localizedDescription] : @"Unknown Error");
              exit(1);
          }
      }
      return 0;
  }

上面mysqlDB类是我自己建的，做的就是方便拿MYSQL_RES的数据而已 你完全可以自己按自己习惯写一个，关键是CoreData中的读写操作。

假设我们现在在xcdatamodeld中创建一个名叫Test的Entity，其中有两个属性：name和age，生成NSManagedObject子类后在main中import “Test.h”

1：创建NSEntityDescription

  Test* test = [NSEntityDescription insertNewObjectForEntityForName: @"Test" inManagedObjectContext: context];

我们通过方法名称大概可以了解，这就是CoreData作insert操作的方法。

2：设置数据

  [test setName:@"abellee"];
  [test setAge:10]; //在xcdatamodeld中设置attribute的类型的时候 如果设置为integer32 那这里就需要使用NSNumber 如果什么都不设置 这里就是int

好了 就这样

  NSError *error = nil;
    if (![context save:&error]) {
        NSLog(@"Error while saving %@", ([error localizedDescription] != nil) ? [error localizedDescription] : @"Unknown Error");
        exit(1);
    }

紧接着的这一步就会把上面设置的数据保存到CoreData中了，也就是sqlite文件中。

然后是有关relationship的使用，我这里直接讲一对多的情况，因为一对一的情况很简单，随便找资料大把。

当你给一个Entity创建一个relationship的时候 你可以勾选To-Many，这样这个relationship就可以包含很多条数据，

例如：一个公司下有很多部门，创建一个Entity叫Company， 再创建另一个Entity叫Department，然后在Company里Add Relationship，把这个relationship指向Department，再勾选属性面板上To-Many，这样，就可以让你一条Company数据里携带多条Department数据。

当生成这两个Entity类后，Company类里会有一个NSSet，就是Department的数据。起初我很疑惑的是，CoreData怎么把我要录入的数据准确的关联在一起，但可能当时脑子发热，因为你会按关系存入数据，CoreData自然会记住这些关系，读的时候自然就有据可寻了。

好了，代码说话：

  #import "Company.h"

  #import "Department.h"

  NSMutableArray* companies = ...; //假定你已经把所有公司的数据读出来了

  //快速枚举

  for(NSMutableArray* entry in companies){

           Company* company = [NSEntityDescription insertNewObjectForEntityForName: @"Company" inManagedObjectContext: context]; //一条NSEntityDescription就会入sqlite中写入一条数据

           //设置公司数据

          [mysql query:[NSString stringWithFormat:@"select * from department where company_id=%d",  [entry objectAtIndex:公司数据id索引位置]]];



           NSMutableSet* mutableSet = [NSMutableSet setWithCapacity:0]; //1：NSMutableSet继承自NSSet。 2：setWithCapacity:0 如果不够，会自动增加容量。

           //读取mysql_store_result数据，转为NSMutableArray* departments

          for(NSMutableArray* row in departments){

                   Department* department = [NSEntityDescription insertNewObjectForEntityForName: @"Department" inManagedObjectContext: context];

                   //设置department中数据

                  //添加到mutableSet中

                 [mutableSet addObject:department];

           }

           [company addDepartment: mutableSet];

  }

这样，CoreData就会自动把公司跟部门的数据存在两个指定的表中去。

在用CoreData的时候，创建Entity都是针对于.xcdatamodeld文件来做的，但不知道为啥我之前创建的这个Model文件变成了Mapping Model文件，也就是.xcdatamodel，然后纠结着想把Mapping Model转到Model，因为里面创建了很多Entity 不可能再重新搞一遍 会疯的- -！所以找google问，万能的stack overflow里查到说可以用Editor->Add Model…来创建，我照做了，但是比如我现在的.xcdatamodel文件名叫a.xcdatamodel，操作后就会多出一个a 2.xcdatamodel 再个一起存在于a.xcdatamodeld下，有洁癖的我怎么能容忍这样的事情发生，然后这样搞搞 那样搞搞，运行总报错，具体什么错误就不多扯了，是NSPersistentStore的一个错误。然后又纠结一番后发现有一个比较SB但完全不用动脑的机械方法可以达到目的。

1：重新创建一个CoreData的Model文件，例如命名成:a.xcdatamodeld

2：在Model文件上右键，点击show in finder

3：在finder的Model文件上再次右角，点击 “显示包内容”

4：里面你就可以找到这个Model文件的Mappding Model文件，也就是a.xcdatamodel，在它上再次右角，还是选择“显示包内容”

5：到这已经是最里层目录了，里面有一个content文件，用文本工具打开，同样的方法打开你事先创建好Entity的Mapping Model文件下的content，把里面所有entry节点，外加底部所有的elements节点都复制到a.xcdatamodeld下的a.xcdatamodel下的content文件里保存，再在Xcode里看一下 你就会发现全部数据都已经转移过去了。。

————————————————————————————————————————

当你把所有要捆绑的数据录完后，在Products下的App上右键，点击Show In Finder，然后你就能看到一个Model文件名的sqlite文件，就它了 复制他，一样操作放到ios应用下就搞定了。
