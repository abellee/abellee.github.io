node-webkit发布后flash无法被调用的坑

继上一篇，继续自挖自跳，在做flash开发的时候，我们都知道，如果swf在本地运行的话，是有信任目录的问题的，也就是说，如果你的swf里有涉及as3访问网络资源之类的，你需要把该swf运行的本地目录地址加入到FlashPlayerTrust目录下才行，同样的，上一篇解决了swf加载的问题，同时也要解决信任目录的问题。而我们打包node-webkit应用后，用户具体安装在哪个目录我们是无法确定的，所以必须要获取应用目录地址，下面是代码：

    var filename = global.module.filename;
    var arr = filename.split("/");
    arr.pop();
    var appDir = arr.join("\\");

这样我们就获取到了应用被安装的目录了，其实这里获取的并非真实的安装地址，因为我们最终打包的node-webkit应用是exe形式的，会被系统解压到一个临时地址，把exe里的文件都解压出来进行执行，所以这里获取的是应用的临时目录地址，也就是说，随着应用的执行，信任目录会越来越多，所以在每次添加之前，最好先清一下之前添加的。

获取了目录地址之后，就要把地址写到flash的可信任目录里，谷歌搜一个nw-flash-trust的nodejs库，代码如下：

    var appName = "testAppName";
    flashTrust = require("nw-flash-trust");
    try{
        var trustManager = flashTrust.initSync(appName);
        var isTrusted = trustManager.isTrusted(path.resolve(appDir, ""));
        if(!isTrusted){
            trustManager.add(path.resolve(appDir, ""));
        }
    }catch(err){

    }

so, that’s it!
